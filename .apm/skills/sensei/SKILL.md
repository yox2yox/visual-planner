# sensei

機能案や移行計画、現在の実装状況を **2 つのファイル**に変換します。`plan.json` と、自己完結した `plan.html`（plan-viewer バンドルに JSON をインライン化したもの）です。ユーザーは HTML ファイルを任意のブラウザで開けます。
プロジェクトのことを全く知らない人が見ても超わかりやすく理解できるような教えるプロの技をまとめたスキルになります。

## スキルペルソナ

このスキルは下記のようなペルソナと状況を想定しています。

ペルソナ: 20年目のベテランエンジニア

想定: 自分のプロジェクトを新卒の経験の浅いメンバーへ引き継ごうとしている

## 使うタイミング

ユーザーが次のような依頼をしたときに、このスキルを使います。

- 実装計画、提案、リファクタリングを可視化したい
- 現状のアーキテクチャを説明して
- 機能におけるコンポーネント間のデータフローを示したい
- 「plan viewer」や「この計画をファイルとして共有できる形」を作りたい

対象外: コード生成、ドキュメント執筆、計画 JSON の生成に帰着しない作業。

## ワークフロー

1. プランを生成する
   plan.schema.jsonの構造に従ってプランを作成する。descriptionをよく読んでその内容に必ず従うこと。

2. **ヘルパースクリプトに JSON を渡してファイルを生成する**。JSON ファイルのパスまたは stdin と、出力 basename を渡します。

   ```bash
   node <skill-dir>/scripts/make_plan.mjs path/to/plan.json /abs/path/to/output-basename
   ```

   または stdin を使います。

   ```bash
   echo "$PLAN_JSON" | node <skill-dir>/scripts/make_plan.mjs - /abs/path/to/output-basename
   ```

   スクリプトは計画を検証し（型、一意な id、parentId/architecture edge 参照など）、次を出力します。
   - `<output-basename>.json` — 整形済みの検証済み計画
   - `<output-basename>.html` — viewer バンドルのコピー。計画 JSON は安全にエスケープされ、`<script id="plan-data" type="application/json">…</script>` にインライン化されます
     絶対パスは stdout に 1 行ずつ出力されます。Node.js が必要ですが、このリポジトリの依存関係として既に使えるため、インストール手順は不要です。

3. **生成したファイルパスをユーザーに渡す**。次の3点をユーザーに渡す。

- htmlファイルパス
- `file://`から始まるhtmlファイルへのリンク
- `file://`から始まるhtmlファイルへの平文URL

## 執筆ルール

プランの生成には下記を**必ず**守りましょう

- プランを読む人はこのシステムのことを全く知りません。知らない変数名、クラス名、関数名など出てきてもわかるように説明しましょう。あなたはそれをこなせる能力があります。
- このシステムに固有の変数名、クラス名、関数名などについては全てに必ずリンクを付けましょう。すぐにヒントが読めるようになります。
- 単語やシーンに関して関連するコードや計画する実装がある場合は必ずコード抜粋を付けましょう。
- 最初に比喩を 1 つ選び、最後まで保ちます。同じ計画の中で「レストラン」「工場」「郵便局」を混ぜません。
- ユーザーが各セクションで最初に読むのが、takeaway(一言で)セクションです。こちらではパッと読んで全体の流れをつかめるように、最初に選んだ比喩の世界の言葉を使ってわかりやすく簡潔に説明しましょう。読む気の失せる長い文章は不要です。

# glossaryリンクの付け方

下記のように、hrefに#glossary:とその対象となるglossaryのIDを付けて指定します。
`<a href="#glossary:ID">label</a>`

## 既存計画の編集

- ユーザーが既存の plan `.json` ファイルを提供した場合は、そのファイルを直接編集し、同じ出力 basename で `make_plan.mjs` を再実行して HTML を再生成します。
- ユーザーが以前の実行で生成した HTML ファイルだけを持っている場合は、`<script id="plan-data" type="application/json">…</script>` ブロックから JSON を抽出します（`<` は `<` としてエスケープされたプレーン JSON で、`JSON.parse` がそのまま扱えます）。その後、生成スクリプトを再実行します。
